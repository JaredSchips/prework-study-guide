HTML

  • <head> contains information about the webpage which does not appear on the webpage itself.
    • <meta/> represents one "setting" for the webpage.
      • charset sets the encoding format for the webpage (best to use UTF-8, or unicode).
      • name and http-equiv represent which setting you want to edit.
      • content The value of this attribute will set the value of the setting specified by the name or http-equiv attribute.
      • It is a good idea in every webpage to ensure the setting name="viewport" has the value of content="width=device-width, initial-scale=1.0". This ensures that the webpage is scaled to fit the size of whatever aspect ratio screen it is being displayed on.
    • <title> represents the text to be displayed at the top of the browser. Is used by search engines.
  • <body> represents the visible content on the webpage.
    • <header> contains the navigation bar and other content which goes on the top of the screen.
    • <main> contains the bulk of the webpage.
      • <section> is used as a a container for sections of <main>.
    • <footer> contains content at the bottom of the page such as the author, copyright, contact information, sitemap, and navigation.

    • <p> represents a paragraph of text on a new line.
    • <br/> represents an empy line.
    • <li> represents a list item. Should be nested within a <ul> or a <ol>.
    • <ul> represents an unordered list (ie bullet points).
    • <ol> represents an ordered list (ie numbered).
    • <h1>, <h2>, <h3>, <h4>, <h5>, <h6> all represent different tiers of headings. only one instance of <h1> should exist per webpage. These are used by search engines.
  • Attributes are additional pieces of customization which can be added to html elements.
    • id an identifier used to find and select elements.
    • class is similar to id. (TBC)

Git

  • git clone SSH-value downloads the repo specified by the SSH-value onto the local computer.
  • git status checks what branch you are currently on.
  • git config --global <setting> <value> changes the value of <setting> to match <value>. If no value is provided, returns the current value. Settings include:
    • user.name: the username publicly associated with each of your actions on Git.
    • email: the email publicly assoicated with each of your actions on Git.
  • git checkout [-b] <branch name> switches the active branch to the specified one.
    • -b tells git to create the branch before switching to it.
  • git pull origin <branch name> updates the GitHub copy of the specified branch onto the local computer.
  • git add [-A] <file-1> <file-2> <file-n> adds the specified files to the staging area of git.
      -A adds all files in the active branch to to staging area.
  • git commit [-m "message"] commits the changes in the staging area to the branch.
    • -m "message" includes the message specified in quotes with the commit.
  • git push origin <branch name> uploads the specified branch to its GitHub copy.
  • git revert <SHA-code> reverts the current branch to the commit specified by its unique SHA code.
